What really survives from old legacy applications? To answer to this question we should try to understand to what every code line is related:

  • Technical, functional and environment infrastructure
  • Functional behaviour
  • Formal input controls
  • Functional input controls logic
  • Data access read
  • Data access update
  • User interface
  • Display logic
  • Update logic

The user display, update and functional input controls logic, is a rate very low compared to the total amount of instructions and should be really the logic to be coded in the new application too; all of other code lines types will be embedded and implicitely defined by the Forward function model.

Amrita is an elementary data driven system focused on data transformations detecting, from user input to database update and from database read to user display.

Re-maker is able to populate the Forward function model, at least to produce an immediate running prototype, that can be than made richer and complete in the Forward model driven development environment.
 

Re-Maker module is the reason for which all Amrita suite has been designed and built and is not a source converter.

In order to remake an old legacy application, it's necessary to master the knowledge of the original legacy application, to be able to extract and formalize the functional requirements in a model, and to own the technology to realize applications by models.

The legacy application knowledge is achieved by the Amrita Assesment process and the technology to realize applications by models is implemented by Amrita Forward, the Amrita Model Driven Development module.

Hence the function model, used by forward as an executable set of analysis declaratives, is the bridge among old legacy applications and modern java applications.

Therefore the main goal of Re-maker is to provide a set of tools to extract all available function informations, in order to populate the function model, a declarative code, not procedural, analysis and event oriented, immediately executable as a Forward application.
 
Re-maker allows you to extract an executable formal analysis function model to an high abstraction level

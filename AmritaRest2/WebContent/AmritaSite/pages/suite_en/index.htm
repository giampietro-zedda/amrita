Starting from old Cobol applications sources, the assesment process extracts detailed quality informations and the application functional model.

With an innovative sources static analysis algoritm, Amrita gets all dynamically originated relationships between objects, making the application knowledge exaustive and detailed.

This is a constraint for any further safe activity of re-engineering or maintenance.

Then, maintenance Amrita modules allow you to maintain old applications with an effort and a skill required, dramatically reduced, with a complete automation of many maintenance requests.

Forward module rapresents the bridge between old legacy applications and modern application written from scratch. The shared point is the function model extracted by assesment process, directly used by Forward in a model driven development mode.

All of e-Amrita suite modules are realized by Forward.
 

Amrita is a sanscrit word that means ambrosia, the nectar that gives immortality, the immutable essence in everything, inside shapes always changing.

This is the spirit of e-Amrita, a valuable effort to find and manage data spreaded inside and thru applications, to find out what really doesn't change with technology: application requirements and elementary data logics.

Amrita, as Automatic, Maintenance, Re_engineering, Integrated, Technology Application is the result of over 25 years of applied research of an italian engineer, Giampietro Zedda.

There are remarkable products on the market to do some tasks faced by Amrita, but Amrita solves they in a plain, integrated, just not only technical way.

Amrita has been designed with the goal of a functional re-engineering of old legacy applications, written in Cobol language, to get new, modern java OO applications.

  
 
 
Reusability of even a single instruction, intrinsec quality, design for changing, this is the e-Amrita challenge

It's possible to share functions, forms,single panels, logical data view to perform database access and specific java methods, with all applications.

The execution and the application behaviour can by strongly configurated at runtime by user login, form, data access, variables content, with further reduction of new applications need.

Furthermore forward applications are naturally internationalizated and all texts can be set by language.
The software development cycle is incremental and there is no difference between the prototype and the final application, because they are using the same models and infrastructure.

Forward implements an elementary data item driven approach, to say that elementary data, with name and type, pilots the application.

Only functional focus is required and technical details are unnecessary.

With Forward you are allowed to realize applications without to escape from the analysis domain.
Of course the application running is java and so, if it needs, all java features are available.

All Amrita modules are realized by Forward.

  The Forward module symbolyzes the bridge bridge between old legacy Cobol applications managed by Amrita modules and modern, new java applications.

Forward is what that today is called an MDD (Model Driven Development) to realize applications by patterns against hard manual expensive code towards a declarative model approach.

Forward is based on a general and straightforward application model, where are depicted in a way effective, exaustive and powerful all application key points: user interface, execution flow, data access and application logic.

Forward is written in java and produces java code. Main aspects to be underlined are the development time and maintenability dramatically reduced, and a reusability at the model and single application logic level.

Reusability is a really key point in Forward applications.

The snippet on the left shows the declarative nature of a Forward application. There are more than 120 ON_EVENT condition and more the 150 DO actions, to satisfy all common functional needs.

When a specific or procedural task must be executed it's possible to call a resusable application method by means of DO_EXEC_METHOD action.
A reusable application method is a java method dynamically called runtime by reflection, with only two parameters: System and Function objects.

System allows you to get and/or to manipulate all current runtime application informations and Function to get the function descriptor, to alter the normal flow and to set the Forward data Model, implementing the MVC pattern..

Because of FSD is an elementary data driven development system, based on pseudo code and declaratives, many normally common errors done are avoided.

Mainly, when a data is not showed, an event is not triggered or a data is not updated, may be that simply you have wrong the name of the elementary data.

Furthermore, may be that the user interface control is not compatible with the data base item with the same name.

Function structure and rules are coded like a pseudo code, organized by not procedural events and actions to perform, called by Forward FSD (function specification and documentation).
To say that what normally it's only documentation and general specification, for Forward is the final application.

Indeed FSD is an object, realized with an original use of Java, without external descriptors, XML configuration or technical skill required.

Taking a look to a Forward application, it's difficult to understand that FSD is java code and not a textual descriptor.

Hence FSD is both a model direcltly executable and an application descriptor and lets to realize the java based cross platform portability.

At the end FSD is a set of declaratives of user interface components and coded events and actions to perform when events occur.

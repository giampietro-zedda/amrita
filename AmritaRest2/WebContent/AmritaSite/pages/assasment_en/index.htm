Amrita Assesment has been designed to rank legacy objects to get exaustive informations directly and only from sources code, by means of specialized static source analyzers and further execution of complex algoritms and processes.

Source types analyzed are Cobol programs batch and Cics, copy books, Jcl Z/OS, Cics Bms, Sql DDL, Sql DML embedded in Cobol programs.

 Assesment is a complex task, realized by a fully scalable operations set, that allows you to run only the process that you really need, with only required processing options.

Any process can be also run subsequently the initial source analysis, to evaluate specific metrics, to solve the dynamic code, to detect the dead code and so on.

All specific Assesment analyzers are much more then specialized sources parsers, as only by a static analysis, they are able to get dynamic informations not directly visible by source, and dynamically generated only at run time.

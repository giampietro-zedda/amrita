Tests management should begin when the function model has been initially coded and the function is running, and continue at every change of the model.

Amrita-Test is the module designed to manage the test phase and, likewise other tools of this category, implements the most common characteristics for a test tool but at the functional level specification.

Amrita-Test lets you to save time in the test phase through the automation of test activities and allows you to certify the correct function execution.

Main available managed features are:

  • Unit test
  • Unit test chains
  • Tests report
  • Tests recording by keystroke
  • Events coverage matrix report
  • Fields masking management
  • Automatic recording tests execution
  • Not regression execution tests
  • Automatic unit test generation
  • Data base of tests done management
  • Automatic unit test update on function model changing
 

All of Amrita modules are structured to turn the focus from application code to the functional logic, from the program structure to the function model, to achieve an higher abstraction level.

The application development by models implemented by Amrita Forward allows you to be focused only on function behavior, by means of straightforward events and actions declarations.

Therefore to test a Forward application means to verify the effect of analysis directives against the expected execution of standard procedural code.

Amrita-Test allows you to check a model, not to test instructions execution.

A minimal forward application test means to check that all events have been verified once and all actions related have been succesfully executed too.

To get a more complete forward application test it's necessary to verifiy all sequences of events. as managed by means of the events coverage matrix,.
 
Test management is not a cost effective phase when is fully integrated in the ordinary prototyping activity

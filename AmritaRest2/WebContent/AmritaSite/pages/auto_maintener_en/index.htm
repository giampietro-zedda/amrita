Hence can be automatically fixed all effects of actions related to the fields dimension changing of database table column, screen fields, interface fields and so on.
All working fields and areas indirectly affected can be fixed too.

There are others maintenance types faced by Auto Maintener that can be an aid for the maintener, to deal with evolutive maintenance, or just a more complex maintenance, not properly related only to the elementary
data item transformations.

Auto Maintener can detect and propose the most probably suitable sources point to modify, but the advise should be carefully verified to detect all consequences.

In any case the time to spend to find out all programs and objects to modify and the probably point to fix, is dramatically reduced.

Typical maintenance of this type are:

  • New Database table/Screen/Interface field
  • Function erasing
  • Program erasing

 

Auto Maintener is able to perform phisically some maintenance tasks, with generation of modified sources, with no manual intervention required.

This module works with informations obtained by Amrita Impact and takes advantage from the assesment process and all application informations obtained, structured and stored.

The available application knowledge includes a wide detail level, from the single coded instruction to all relationships between objects, even when originated by dynamic code.

The maintenance process can't be a completely automatic activity but many maintenance types can be done by Auto Maintener, with automatic source update, just confirming an update proposal preview to perform.

Amrita-Doc makes easy to get a program documentation card with all available and required informations.

Amrita-Doc takes advantage of all informations obtained by the assesment process and by amrita modules,
to show and export data about application knowledge,
with three main guidelines:
  • Selective objects to document
  • Selective informations type to expose
  • Required detail level from general to particular

To get organized informations about:

  • Sources
  • Programs relationships and characteristics
  • Records layout & Sql DDL informations
  • Quality informations
  • Functional behaviour
  • Copy used
  • Etc.

You are allowed to print showed informations or to produce a structured PDF with all extracted informations.
 


Traditional documentation was realized by lists, documents and reports as an application photo at a specific date, unfortunately probably not up-to-date just few days later.


We are speaking of technical documentation, functional characteristics and generally of any
kind of information related to the source code.

Of course is not possible to get business rules directly from code but many business rules have an effect on sources codes, that the Amrita assesment process is able to detect.

We can say that all Amrita system could be considered a powerful documentation system too because, through any specific module, you are allowed to get all informations you wish just only when you need.

In order to achieve this result, the Amrita assesment process should be executed when any source has been modified, ensuring Amrita-Doc to produce always correct informations.
 
Technical documentation becomes not necessary when all data are automatically available with no extra cost

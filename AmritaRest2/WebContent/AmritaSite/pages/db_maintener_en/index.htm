A logical data view is based on an Amrita technical layer that implements an ORM like Hibernate.

All informations to be used to realize any automatic database access, are coded by annotations in each java pojo class related to Sql tables.

Because of the initial step to build Forward running data base oriented applications is data access, Db Maintener allows you to code and test separately all necessary logical data view.

A Logical data view realizes automatically a structured query that can be coded by a complex set of Sql Inner Join.

Db Maintener supports you on activities like:

  • Automatic entity definition and entity been class source generation
  • Logical data view creation wizard
  • Logical data view testing
  • Sql DDL generation and export
  • Selective Sql data export/import
  • Database functional support utilities
 

Db Maintener is the natural complement to be used developing applications with Amrita Forward.

It isn't a general purpose tool to manage a data base, but all available features are to be considered a support to easily define and manage the data access and the persistence of Forward applications.

It's often necessary, developing modern applications, to use external tools and environments to do ordinary operations like Sql DDL creations, building and testing structured queries, export and import selective data and so on.

The aim of Db Maintener is to integrate this kind of activities in a simple set of features with a functionally oriented approach, against a technical, data base dependent, point of view.

In a Forward application the data base access and the persistence are realized by a specific infrastructural Amrita software layer named Logical Data View.

 
Data access abstraction at the logical data view layer is the pivot in the forward applications architecture

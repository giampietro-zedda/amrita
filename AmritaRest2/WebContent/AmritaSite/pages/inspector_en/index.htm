This kind of informations are about how the program is structured, the interaction with programs called and caller with related shared data, with contextual analysis of execution paths, and data modified.

Inspector doesn't work with original sources but with the java object descriptor generated by means of the Assesment process and the Amrita database populated by the Assesment process.

This means that all of informations are already available in a structured form, designed for maintenance purposes too.

For about generalized dynamic instructions, solved by initial source analysis or by subsequent Assesment processes, all possible dynamic values, detected by static analysis, are shown for each dynamic instruction.

For example, besides a "call nome-pgm" instruction, are listed all values that the field nome-pgm could have runtime, to say all programs potentially called by means of the dynamic instruction.

The dynamic instruction code solved by Amrita specialized analyzers, remarkable from the Assesment and quality control point of view, is essential in this maintenance phase, to avoid a not desiderable regression test fault.
 

In the maintenance process, it's not enaugh to know all objects to be modified and the relationships among objects involved.

As underlined, Amrita Viewer allows you to get this level of understandig solving about the 50 percent of the entire maintenance process.

Afterward it's necessary a further comprension level and more detailed and careful informations, in order to actually modify source codes.

Always old Ibm statistics show that this maintenance phase can be up to 20 percent of the overall maintenance time.

This is the goal of Inspector, to reduce significantly the time spent to acquire all technical details necessary to make code changing affordable and safe.

Furthermore, in this phase the risk to make the application instable and defect prone is high.
Inspector allows you to manage, to get a set of technical level informations.

  
 
 
An error prone maintenance does require to master all programs technical details too

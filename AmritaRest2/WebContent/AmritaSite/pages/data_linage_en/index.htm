Furthermore, often the name convenction is not related to a functional classification but to a specific company organization and the application logics are spreaded across different systems and subsystems.

The maintenance process, then, needs to get sets of objects with specific spreaded functionalities or to get boundary and shared objects.

Amrita Data Linage deals just with this kind of concerns, realizing the goal to detect a set of object names matching a complex sequence of constraints.
The main features of Amrita Data Linage to get a set of objects names are:

  • All objects of a system/subsystem or referencing a specified system/subsystem
  • Names conventions, source and status properties
  • Source dimension and quality measures values
  • Source contents as fields, specific instructions, code sections and so on
  • Objects types and options like OBJECT_PGM, OBJECT_ENTITY, PGM_SQL, PGM_WITH_MAP.
  • Access/Non access to entities in read/update/insert/delete
  • read/update/insert or not to soecific entity fields
  • Relationships with other objects like PGM_CALLS_PGM and so on
 

Legacy applications get the functional compliance by means of code spreaded across many programs and thousand of code lines.

Thanks to design time source objects name conventions, you are often allowed to identify all objects involved in a specific functional area or related to a specific business rule.

Nevertheless the maintenance process, year after year, decrease the quality of applications through exceptions and out of context objects with a progressive reduction of the original design effectiveness, making the names convention less reliable.

The first phase of the Amrita Assessment process is the Source Acquiring and so, through a custom name based rule system, all objects are classified by two macro areas, a system and a subsystem, where the meaning is customer dependent.

Any object is identified by a system and subsystem owner and, when an object is referenced by a system or subsystem not owner, Amrita marks the object in the system/subsystem referencing too.
